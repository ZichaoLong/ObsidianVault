# Learned Noisy Directional Access 备忘

本页是独立备忘，不合入当前主线。它整理 `active foveated workspace access` 背后的一个更硬抽象：

> 模型能否从局部观察中学习一个带噪的方向判别器，并用它把全局搜索变成逐步缩小候选空间的主动搜索？

这不是 `Load/Store` 的证明，也不是 `O(log n)` 的无条件证明。它是 B 分支下一个候选理论桥：把“人眼扫描 / 移动视野 / 放大缩小 / 局部判断”的直觉，压成可实验、可失败的问题。

## 核心判断

`active foveated workspace access` 的核心不是“局部读取”本身，而是 `可导航结构 + 局部方向信号`。

- 如果待查看数据没有任何可利用序关系，这个方法没有特殊优势。
- 如果序关系确定、可形式化、可程序化，临时生成查找代码、二分、索引、SQL、LSP、tree-sitter 或专用 resolver 会成为强对手。
- 这个方法的发挥空间在中间灰色地带：状态空间有局部或全局序关系，但这种序关系不够干净，不能直接写成稳定确定程序，需要模型基于局部观察做方向判断。

这里用 `可导航结构` 比 `顺序结构` 更准确。它包括一维顺序，但不止一维顺序：

- 二维或三维空间。
- 时间线。
- 层级树。
- 调用图和依赖图。
- 论证结构。
- agent trace 的状态演化链。
- 由 bookmark、摘要、分段、索引或验证点主动构造出的拓扑。

因此，直接牵引 TapeWalker 的命题不是“序关系普遍存在”，而是：

> 自然任务分布中广泛存在或可被构造出可导航结构；模型可能学习到带噪方向判断策略，在局部观察预算下逐步逼近目标。

务虚价值：

- 它像一种通用回退机制。当没有现成 key、schema、index 或 resolver 时，系统仍可以通过移动视野、局部观察、标记和逐步修正工作。
- 现实任务似乎常常不是完全无序的：具身世界有空间连续性，物理世界有因果和时间连续性，长文本有论证推进，代码有调用栈和依赖图，agent trace 有状态演化顺序。

务实裁决：

> 最终仍要看能否找到有说服力的任务。任务越强，机制越有价值；任务越弱，它就退回为普通研究或 TapeWalker 实验 scaffold。

## 方向判别器与复杂度

一个更精确的问题是：

> 每次观察一个局部窗口后，模型能否以高于随机的稳定准确率判断目标在左 / 右、上 / 下、内 / 外、前 / 后？

如果答案为是，且满足若干条件，就会接近 `noisy binary search` / `probabilistic bisection` / `generalized binary search`：

- 每次查询能把候选空间大致均分。
- 方向回答正确率稳定为 `p = 1/2 + gamma`。
- 错误不是系统性偏向。
- 系统有验证、回退、重复采样或贝叶斯更新机制，避免一次错判毁掉整条路径。

在这种模型下，查询次数可以保持对 `n` 的对数依赖，但常数随噪声快速恶化。粗略可以理解为：

```text
queries ~= O((log n + log(1/delta)) / gamma^2)
```

其中 `delta` 是失败概率，`gamma` 是高于随机的 margin。

这给出两个边界。

第一，这不是确定性 `O(log n)`。只有方向 oracle 总是正确，或错误可以被确定性验证并纠正时，才接近确定性二分。

第二，如果 `gamma` 很小，或随着长度、任务难度、分布迁移而下降，复杂度优势会迅速消失。若局部窗口无法提供方向信号，最坏情况仍接近线性扫描。

因此，TapeWalker 式实验不应直接声称 `log(n)`，而应测量：

- 局部窗口到方向判断的准确率。
- 每次观察带来的信息增益。
- 错误方向判断是否可恢复。
- 方向准确率是否随 workspace 长度保持。
- 方向策略能否迁移到 heldout 任务或更长长度。

## 通用任务与结构学习

不能说模型能在任意任务中学出有效序关系。

对任意任务分布，没有免费午餐。如果任务是完全任意或对抗性构造，任何访问策略都不能普遍优于随机或线性扫描。

但现实任务不是均匀随机任务。很多自然任务分布有强归纳偏置：

- 空间连续性。
- 时间连续性。
- 因果顺序。
- 论证推进。
- 调用栈、控制流和依赖图。
- 状态演化顺序。
- 错误传播链条。

所以更可守的命题不是：

> 通用任务都能学出序关系。

而是：

> 在某些自然任务分布中，模型能否通过训练学到可迁移的主动观察策略，或通过运行时标记、摘要、分层和验证点构造可导航结构，使局部观察持续提供正信息增益？

这个命题不要求存在显式可算的确定性序关系，但要求任务分布中存在可学习的统计结构。

## IP = PSPACE 的位置

`IP = PSPACE` 不是 TapeWalker / `learned noisy directional access` 的直接哲学牵引。

它更适合放在远层背景：它说明复杂对象在合适协议和表示下可能存在可被较弱 verifier 局部随机检查的结构，但它不说明这些结构天然可导航，也不说明模型能通过移动视野把结构找出来。

关于 `IP = PSPACE`，一个较准确的表述是：

> prover 的计算能力可以极强；即便问题求解本身可能很难，仍可能存在一种交互协议，使得较弱的 randomized verifier 通过局部查询、随机抽查和一致性检查，高概率验证 prover 给出的结构化声明。

这确实支持一种哲学牵引：

> 复杂对象中可能广泛存在可被交互、随机化、局部一致性检查暴露出来的可验证结构。

但需要加三层限制。

第一，它指向的是 `可验证结构`，不是 `可发现结构`。

IP 里的 prover 可以非常强。定理不说明 verifier 能独立找到证明，也不说明 agent 能靠局部扫描自己发现所有结构。

第二，它指向的是 `协议化 / 编码后的结构`，不是自然表面形态中自动显现的结构。

IP = PSPACE 的证明依赖 arithmetization、交互协议和随机一致性检查。结构不是原样躺在输入里，而是在恰当表示和协议下被构造出来。

第三，它不表示复杂度消失。

它说明“强求解者给出结构化声明时，弱验证者可以高效验证”的边界非常大；它不说明 PSPACE 问题可以被一般多项式时间求解。

因此，用户提出的说法：

> 既然可用更小复杂度验证，那么说明有某种结构的存在；这个定理指向的是“结构的普遍存在性”，而非“复杂度的消失”。

整体方向是准确的，但建议改成更克制版本：

> `IP = PSPACE` 不意味着复杂问题容易求解；它说明在合适的交互协议和表示下，即使非常复杂的计算也可能具有可被较弱 verifier 通过局部随机检查验证的结构。这可作为“结构可能广泛存在、可通过交互暴露”的哲学牵引，但不能直接推出现实任务中存在可由模型主动扫描发现的序关系。

## 与控制反馈线的关系

需要区分三层牵引。

第一层，直接牵引 TapeWalker / `learned noisy directional access` 的，是 `可导航结构`：

> 自然任务分布中广泛存在或可被构造出可导航结构；模型可能学习到带噪方向判断策略，在局部观察预算下逐步逼近目标。

第二层，牵引 A+B 合流的，是 `可交互验证结构`：

> 复杂任务不一定只能靠一次性全局展开；交互、局部检查、随机抽查、状态化声明和验证协议，可能把复杂结构暴露给较弱控制器。

第三层，`IP = PSPACE` 只适合做远层背景：

> 它支持“合适协议下可验证结构可能非常广泛”这个哲学直觉，但不能直接支持“TapeWalker 可通过局部移动发现结构”。

因此，`IP = PSPACE` 若放入历史动机，最合适的位置不是作为 `Load/Store` 或 `active foveated access` 的理论证明，而是作为更弱的旁支脚注：

> 复杂任务不一定只能靠一次性全局展开；交互、局部检查、随机抽查、状态化声明和验证协议，可能把复杂结构暴露给较弱控制器。

这与控制反馈线的关系可以这样组织：

- B 分支负责主动选择观察位置：看哪里、缩放到什么粒度、如何移动、何时回退。
- A 分支负责让观察、声明、验证、提交、回滚成为显式状态事件。
- A+B 合流时，系统才更接近“可交互验证、可局部纠偏的结构挖掘系统”。

因此，`IP = PSPACE` 适合作为历史动机中的旁支脚注或哲学备忘，不适合作为主线中的硬支撑；更不应作为 TapeWalker 式主动视野访问的直接动机。TapeWalker 的直接动机应来自可导航结构、带噪方向判断和主动观察策略。

## 建议实验切口

最小实验应直接测 `learned noisy directional access`，而不是泛泛比较“有无 TapeWalker”。

### 人工任务

构造一维或二维 workspace：

- 目标位置由隐藏规则决定。
- 局部窗口含有带噪方向线索。
- 噪声、窗口大小、长度、方向 margin 可控。
- 可以有单调版、非单调但局部可判别版、无方向信号版。

目标：

- 验证模型能否学到方向策略。
- 测长度外推曲线。
- 测 `gamma` 变化对访问步数的影响。
- 用无方向信号任务做负控。

### 半真实任务

优先考虑：

- 长 agent trace 中定位首次错误决策。
- dynamic workspace recovery：任务中断 / compact 后找回当前进度。
- 长文档中定位定义变化、观点转折、约束漂移。
- 日志或执行轨迹中定位首次 invariant break，但没有稳定关键词。

这些任务比普通 code search、JSON 查询、SQL 查询更适合，因为它们有序但不一定可程序化。

### 强基线

必须对标：

- full context。
- random / linear scan。
- generated deterministic search code。
- grep / BM25 / vector retrieval。
- SQL / index / LSP / tree-sitter，若任务适用。
- RLM / recursive context management，若任务是长上下文。

### 主指标

- 方向判断准确率。
- 每步信息增益。
- 观察 token。
- tool calls。
- 定位步数。
- false negative rate。
- 错误方向判断后的恢复率。
- 长度外推曲线。
- selector 轨迹是否能用于监督训练。

## 失败与降级

这条线的失败条件应明确。

- 如果加入 `grep / BM25 / vector / SQL / LSP / RLM / generated search code` 后优势消失，应承认它被现有 Agent 工程吸收。
- 如果只在人工有序玩具任务上有效，结论应限于机制 sanity check。
- 如果方向准确率随长度或分布迁移崩掉，不能声称存在稳定访问复杂度优势。
- 如果任务的“方向信号”由研究者手工写入，必须计入 scaffold 成本。
- 如果无法产生高质量 selector 轨迹，训练数据飞轮价值也不足。

当前最可守判断：

> `learned noisy directional access` 是 B 分支里最有理论味道的候选机制。它的赌注是：现实任务中存在大量“不可轻易程序化、但存在或可构造可导航结构，且局部观察有方向信号”的灰区。如果找不到这样的任务，它应降级为 TapeWalker 的历史动机和实验 scaffold。

## 参考方向

- `Noisy Binary Search` / `Probabilistic Bisection`：带噪方向回答下的搜索复杂度。
- `Generalized Binary Search` / `Active Learning`：通过主动查询减少假设空间。
- `Recurrent Models of Visual Attention`：视觉 foveation 与主动 glimpse 策略。
- `No Free Lunch`：任意任务分布下不存在普遍优势。
- `IP = PSPACE`：交互、随机检查和可验证结构的远层背景，不是 TapeWalker 的直接牵引。
